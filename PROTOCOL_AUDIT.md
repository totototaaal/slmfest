# Аудит соответствия приложения протоколу Kaspi.kz

Дата проверки: 12.05.2026

## ✅ РЕАЛИЗОВАННЫЕ ТРЕБОВАНИЯ ПРОТОКОЛА

### 1. Требования к интерфейсу провайдера
- [x] **HTTPS** — проверка в `security.validate_https()`
- [x] **IP-адреса** — валидация из конфига: `194.187.247.152`, `194.187.245.108`, `197.187.244.108`
- [x] **GET параметры** — обработка через `request.query_params`
- [x] **JSON и XML форматы** — оба поддерживаются в `_build_response()`
- [x] **UTF-8 кодировка** — явно указана в media_type

### 2. Параметры запроса
- [x] `command` — "check" или "pay" (валидация в `process()`)
- [x] `txn_id` — до 18 символов, целое число (парсинг в `_parse_txn_id()`)
- [x] `txn_date` — формат ГГГГММДДЧЧММСС только для "pay" (парсинг в `_parse_txn_date()`)
- [x] `account` — до 200 символов, валидация по regex (в `validate_account()`)
- [x] `sum` — дробное число с точностью до сотых (парсинг в `_parse_sum()`)

### 3. Структура ответа
- [x] **XML формат** — полная поддержка с корректной структурой
- [x] **JSON формат** — полная поддержка
- [x] **txn_id** — включен в оба формата
- [x] **prv_txn / prv_txn_id** — для команды "pay" (id платежа в БД)
- [x] **sum** — возвращается с точностью до сотых
- [x] **result** — коды 0-5
- [x] **comment** — опциональное поле с комментарием
- [x] **fields** — поддержка дополнительных полей (field1, field2, fieldN)

### 4. Коды результатов
- [x] **0** — абонент/счет найден и доступен ✓
- [x] **1** — абонент/счет не найден ✓
- [x] **2** — заказ отменен ✓
- [x] **3** — заказ уже оплачен ✓
- [x] **4** — платеж в обработке ✓
- [x] **5** — ошибка провайдера ✓

### 5. Бизнес-логика
- [x] **Проверка дубликатов txn_id** — `txn_id` unique в таблице `payments`
- [x] **Повторный запрос** — возвращает результат предыдущей обработки
- [x] **Двухэтапная обработка** — команды "check" и "pay"
- [x] **Проверка суммы** — в "pay" проверяется соответствие balance_due
- [x] **Изменение статуса** — account.status и balance_due обновляются
- [x] **Логирование** — все запросы логируются в таблицу `request_logs`
- [x] **Транзакции БД** — ACID гарантии через SQLAlchemy

### 6. Безопасность
- [x] **IP-whitelist** — проверка в `validate_request_origin()`
- [x] **HTTPS requirement** — проверка в `validate_https()`
- [x] **Валидация account** — regex pattern из конфига
- [x] **Блокировка записей** — `.with_for_update()` для concurrency

## ⚠️ ПОТЕНЦИАЛЬНЫЕ УЛУЧШЕНИЯ

### 1. Таймаут ответа (требование: 15 секунд)
**Статус:** Не явно реализовано
- FastAPI имеет встроенную поддержку таймаутов на уровне ASGI
- Рекомендуется добавить явный таймаут для long-running операций
- Можно добавить middleware или использовать `asyncio.wait_for()`

### 2. Обработка дополнительных параметров data1, data2, ..., dataN
**Статус:** Передаются, но не документированы
- Параметры передаются в `params` и логируются
- Протокол не требует специальной обработки, только передачи
- Рекомендуется добавить документацию в код

### 3. Многопоточность (требование: 10-15 одновременных соединений)
**Статус:** Поддерживается автоматически
- FastAPI с uvicorn по умолчанию:
  - Использует asyncio для handle множественных соединений
  - Параллельная обработка встроена
  - Для оптимизации: можно увеличить `--workers` и `--backlog` в uvicorn

### 4. Следование JSON структуре для fields
**Статус:** Корректна, соответствует протоколу
